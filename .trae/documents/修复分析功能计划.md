
# 修复分析功能不调用接口问题 - 实施计划

## 问题分析

当前代码问题：
1. **手动输入提交**：`submitDescription` 直接跳转到结果页，没有调用 `startAnalyze`
2. **图片预览点击**：`handlePreviewClick` 调用了 `startAnalyze`，但可能点击事件绑定有问题

## 修复方案

### 修复点1：修改 submitDescription 方法
```typescript
// 提交文字描述
const submitDescription = () => {
  if (!foodDescription.value.trim()) return

  showInputModal.value = false
  // 调用分析而不是直接跳转
  startAnalyze()
}
```

### 修复点2：确保预览区域点击事件正确绑定
在模板中确保 `@tap="handlePreviewClick"` 正确绑定

### 修复点3：startAnalyze 逻辑调整
```typescript
// 开始分析
const startAnalyze = async () => {
  // 同时支持图片 URL 和文字描述
  if (!uploadedImageUrl.value && !foodDescription.value.trim()) return

  uni.showLoading({ title: '分析中...' })
  try {
    const { data, error } = await supabase.functions.invoke('analyze-food', {
      body: {
        imageUrl: uploadedImageUrl.value || null,
        description: foodDescription.value.trim() || null,
      },
    })
    // ... 其余逻辑
  }
}
```
