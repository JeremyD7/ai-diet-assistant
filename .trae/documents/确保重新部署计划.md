
# 确保 Edge Function 正确部署和调试 - 实施计划

## 问题分析

从日志看：
1. Edge Function 收到了请求（POST 200）
2. 返回了 500 错误
3. **但是没有显示我们添加的 console.log 日志**
4. 这说明 Edge Function 可能没有被重新部署，或者部署的是旧版本

## 可能的问题

1. **未重新部署** - 代码改了但没有执行 `supabase functions deploy`
2. **部署失败** - 部署过程出错
3. **日志延迟** - Supabase 日志有延迟，还没显示新日志
4. **环境变量问题** - SILICONFLOW_API_KEY 未配置或配置错误

## 修复方案

### 步骤1：确认本地代码已更新
检查 `supabase/functions/analyze-food/index.ts` 确认包含：
- console.log("收到请求参数:")
- console.log("API Key 存在，长度:")
- console.log("调用硅基流动 API")

### 步骤2：添加早期日志
在函数最开始添加日志，确保能捕获到：
```typescript
console.log("===== Edge Function 开始执行 =====");
```

### 步骤3：重新部署
使用命令重新部署：
```bash
supabase functions deploy analyze-food
```

### 步骤4：测试并查看完整日志
1. 测试："一碗白米饭"
2. 在 Supabase Dashboard 查看 Edge Function Logs
3. 确保能看到 "===== Edge Function 开始执行 =====" 日志
4. 如果没有这行日志，说明部署失败

## 备选方案

如果重新部署后仍然失败，可以：
1. 使用 Supabase CLI 直接调用函数测试
2. 检查 Supabase Dashboard 中的环境变量配置
3. 尝试直接在硅基流动平台测试 API Key
