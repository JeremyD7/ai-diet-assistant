
# AI 饮食助手 - 登录页和首页设计计划

## 1. 项目调研结论
- 这是一个基于 **uni-app** 框架的跨平台项目
- 使用 **Vue** 语法和 **uni-app 组件**
- 已集成 **Supabase** 作为后端服务
- 当前只有一个简单的 index 页面

## 2. 设计理念与美学方向

### 设计风格：清新自然·健康生活
- **色调**: 以自然绿色为主色调 (#4CAF50, #81C784)，搭配柔和的橙色 (#FF9800) 和暖白色
- **视觉语言**: 有机圆润的设计，柔和的渐变，自然的元素
- **动效**: 流畅的入场动画，细腻的微交互
- **字体**: 现代无衬线字体，清晰易读

## 3. 文件和模块规划

### 3.1 需要创建/修改的文件

| 文件名 | 路径 | 说明 |
|--------|------|------|
| login.vue | `pages/login/login.vue` | 登录页面 |
| index.vue | `pages/index/index.vue` | 首页 (重写) |
| pages.json | `pages.json` | 配置页面路由 |
| App.vue | `App.vue` | 全局样式和配置 |
| (可选) components | `components/` | 可复用组件 |

## 4. 实施步骤

### 步骤 1: 配置页面路由
- 修改 `pages.json`，添加登录页面配置
- 设置登录页为启动页（或根据需要调整）

### 步骤 2: 设计登录页面
- 创建 `pages/login/login.vue`
- 实现以下功能模块：
  - 品牌 Logo 和标语区域
  - 邮箱/手机号输入框
  - 密码输入框
  - 登录按钮
  - 注册/忘记密码链接
  - 社交登录选项（可选）
- 设计特色：
  - 顶部装饰性图形元素
  - 柔和的渐变背景
  - 流畅的输入框焦点动画
  - 按钮悬停/点击反馈

### 步骤 3: 设计首页
- 重写 `pages/index/index.vue`
- 实现以下功能区域：
  - 顶部问候/用户信息区域
  - 今日营养摄入概览卡片
  - 快速操作入口（记录饮食、查看报告等）
  - 今日推荐餐食
  - 底部导航栏
- 设计特色：
  - 数据可视化卡片
  - 卡片式布局，层次分明
  - 滚动时的视差效果
  - 图标与文字的和谐搭配

### 步骤 4: 全局样式优化
- 修改 `App.vue`，添加全局 CSS 变量
- 统一主题色、字体、间距等设计规范

## 5. 技术实现要点

### 5.1 使用 uni-app 组件
- `view`、`text`、`image` 等基础组件
- `input`、`button` 等表单组件
- `scroll-view` 用于可滚动区域

### 5.2 样式方案
- 使用 **uni-app 的 rpx 单位** 适配多端
- CSS 变量定义主题色
- Flexbox 布局为主

### 5.3 动效实现
- CSS `transition` 和 `animation`
- uni-app 的 `uni.createAnimation` API（如需复杂动画）

## 6. 风险与注意事项

### 潜在风险
- uni-app 跨平台兼容性：不同平台（H5、小程序、App）的样式表现可能有差异
- 性能：过多动画可能影响低端设备体验

### 应对策略
- 使用 uni-app 官方推荐的组件和写法
- 动画效果适度，优先保证流畅性
- 测试重点关注主要平台（H5/微信小程序）

## 7. 交付结果
- 完整的登录页面（UI 完整，暂不接接口）
- 完整的首页（UI 完整，数据为模拟数据）
- 统一的设计风格和视觉语言
