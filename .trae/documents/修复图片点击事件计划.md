
# 修复图片点击不触发分析 - 实施计划

## 问题分析

当前代码问题：
1. **预览区域缺少点击事件**：`<view class="preview-wrapper">` 没有 `@tap` 事件
2. 虽然显示了"点击开始分析"的提示文字，但这个文字没有绑定点击事件
3. 清除按钮 (`preview-overlay`) 有 `@tap.stop="clearPreview"`，但图片区域本身没有点击事件

## 修复方案

### 修改预览区域添加点击事件
```html
<view class="preview-wrapper" @tap="handlePreviewClick">
  <image ... />
  <view class="preview-overlay" @tap.stop="clearPreview">
    <text class="clear-icon">✕</text>
  </view>
</view>
```

这样：
- 点击图片区域 → 触发 `handlePreviewClick` → 调用 `startAnalyze`
- 点击清除按钮 → 触发 `clearPreview` → 清除预览（`@tap.stop` 阻止事件冒泡）
