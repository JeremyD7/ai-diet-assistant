# 合并登录注册页面 - 实施计划

## 📋 需求分析
将登录和注册功能合并到同一个页面，通过标签切换显示隐藏，提升用户体验，减少页面跳转。

## 📁 项目现状
- 已有独立的登录页面 `pages/login/login.vue`
- 已有独立的注册页面 `pages/register/register.vue`
- `pages.json` 中分别配置了两个页面路由
- 使用 UniApp + Vue.js 开发

## 🎯 设计理念
- **简洁现代**：清新的绿色主题延续健康概念
- **直观交互**：顶部标签切换，流畅的动画过渡
- **视觉层次**：活跃标签高亮显示，清晰的状态反馈
- **统一风格**：保持现有页面的设计语言和配色方案

## 🔧 实施步骤

### 步骤 1：重构登录页面
- 保留 `pages/login/login.vue` 作为合并后的唯一入口页面
- 移除原来的注册跳转链接
- 添加登录/注册切换标签
- 实现两个表单的条件渲染（使用 `v-show`）
- 合并两个页面的表单数据和方法

### 步骤 2：更新路由配置
- 从 `pages.json` 中删除注册页面的路由配置
- 保留登录页面作为入口页面

### 步骤 3：实现切换动画
- 添加标签切换的状态管理
- 实现表单切换时的过渡动画效果
- 优化用户交互反馈

### 步骤 4：测试验证
- 确保登录功能正常工作
- 确保注册功能正常工作
- 确保切换动画流畅
- 确保表单验证逻辑完整

## 📝 技术细节

### 核心组件结构
```
pages/login/login.vue
├── Header（Logo + App名称）
├── Tabs（登录/注册切换）
├── Form Container（条件渲染）
│   ├── 登录表单（默认显示）
│   └── 注册表单（切换后显示）
└── Footer（社交登录 + 统一底部）
```

### 数据合并
- 登录表单：email, password, rememberMe
- 注册表单：email, phone, password, confirmPassword, agreeTerms
- 共享状态：activeTab, loading状态

### 交互流程
1. 默认显示登录表单
2. 点击"注册"标签 → 平滑切换到注册表单
3. 点击"登录"标签 → 平滑切换回登录表单
4. 表单提交逻辑保持不变

- 统一的输入框样式和交互反馈
